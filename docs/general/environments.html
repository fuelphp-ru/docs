Среды

Поддержка сред помогает FuelPHP и вашему приложению принимать решения на основе настроек среды. FuelPHP сам использует настройки среды, чтобы загружать/переписывать дополнительные настройки конфигурации исходя из того, какая среда активна в данный момент.

Список сред

FuelPHP имеет четыре встроенных среды.

  • \Fuel::DEVELOPMENT
    Среда разработки.
  • \Fuel::TEST
    Среда тестирования.
  • \Fuel::STAGE
    Среда переноса.
  • \Fuel::PRODUCTION
    Среда производства.

Настройка вашей среды

Настройка вашей среды выполняется путем установки серверной переменной FUEL_ENV или путем ручного изменения настроек в fuel/app/bootstrap.php. 

// Внутри app/bootstrap.php
			
/**
 * Ваша среда.  Может быть установлена на любую из следующих:
 *
 * Fuel::DEVELOPMENT
 * Fuel::TEST
 * Fuel::STAGE
 * Fuel::PRODUCTION
 */
Fuel::$env = (isset($_SERVER['FUEL_ENV']) ? $_SERVER['FUEL_ENV'] : Fuel::DEVELOPMENT);

Среды и конфигурация

Исходя из среды, на которую настроено приложение, класс конфигурации ищет связанные со средой файлы конфигурации. Класс конфигурациибудет искать такое же имя файла конфигурации в каталоге, названном именем текущей среды. Пример ниже иллюстрирует это: 

app/
  config/
    stuff.php
      development/
        stuff.php
      production/
        stuff.php

Когда ваша среда установлена на \Fuel::DEVELOPMENT, настройки из stuff.php объединяются с development/stuff.php. Если параметр среды установлен в любой, отличный от \Fuel::DEVELOPMENT, дополнительный файл конфигурации не загружается. Это же верно для любого другого параметра среды, который вы можете использовать.

Практический пример этого - конфигурация базы данных. Отсутствуют параметры конфигкрации по умолчанию (это может быть очень опасным). Есть только связанные со средой параметры конфигурации.
