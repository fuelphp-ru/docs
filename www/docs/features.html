Почему FuelPHP?

HMVC

FuelPHP построен на MVC (Model-View-Controller) паттерне, чтобы иметь полную поддержку HMVC как части архитектуры. Но мы не остановились на этом, мы также добавили ViewModels (также известный как представление модели), которое даёт возможность добавить мощный слой между контроллером и представлением делая их более независимыми.

  • Полная поддержка HMVC, везде и всюду!
  • ViewModels - прослойка между контроллером и представлением. Для большей гибкости.
  • Более гибкая поддержка роутинга (с версии v1.1)

Модульный и легко расширяемый

Почти каждый класс в ядре FuelPHP может быть расширен без необходимости переписывать код, в тех местах где он используется. Вы можете создавать свои собственные пакеты/классы, которые расширяют или заменяют пакеты FuelPHP ядра. Вы можете делать свои приложения модульными. Пакеты и модули позволяют повторно использовать код и предоставляют большую свободу архитектуре самого приложения позволяя применять различные паттерны.

  • Легкое расширение или замена классов ядра
  • Расширение стандартного функционала за счет Пакетов(Packges)
  • Создание модульных приложений, свобода архитектуры, применение паттернов программирования, повторное использование кода

Безопасность

Безопасность была в центре внимания с самого начала разработки Fuel. Fuel обладает высочайшей степенью безопасности. Подробнее о безопасности в FuelPHP можно прочитать в документации.

  • Input фильтрация
  • XSS фильтрация
  • Output фильтрация
  • Защита от CSRF
  • Защита от любых SQL иньекций

Oil: мощная утилита коммандной строки(опционально)

FuelPHP использует возможности командной строки с помощью утилиты под названием Oil. Oil предназначена, чтобы ускорить разработку проекта, повысить эффективность тестирования и отладки, будучи совершенно необязательной для противников использования коммандной строки.

  • Генерация кода, Скафолдинг и генерация админки
  • Запуск миграций Баз Данных
  • Интерактивный дебаг
  • Задания(Tasks) - например когда нужно запускать какой-то фоновый процесс и т.д.

Базовые классы для Контроллеров и Моделей

FuelPHP содержит несколько классов для быстрого старта.

  • Controller_Template - Добавляет функции шаблонизатора в ваших контроллерах
  • Controller_Rest - Это простейший способ создания RESTful API
  • Controller_Hybrid - Комбинация rest и temlate функционала в одном контроллере
  • Model_Crud - Поддержка всех методов CRUD.

Мощный и в тоже время легкий ORM

Он не должен быть заменой Доктрине/Propel, это более легкое решение, но с мощным функционалом способным выполнить очень многие задачи.

  • ООП подход при работе с базами данных
  • Выборка отношения отношения отношения ... и т.д. .. за один раз ("вложенные отношения")

The API of the ORM model is quite similar to that of the Model_Crud base class, which makes switching to ORM when you need extra functionality a fairly painless operation.

Поддержка аутнтификации

Невозможно создать библиотеку Auth, которая бы удовлетворила всех. Поэтому мы вынесли функциональную начинку в различные драйверы аутентификации. Это позволяет одним разработчикам создавать и делиться своими собственными реализациями, а другим использовать уже готовое. Например простой набор драйверов ("SimpleAuth") хорошо подходит для быстрого старта.

  • Содержит простую аутентификацию для пользователей, групп и ACL(Access Control List или ACL — список контроля доступа)
  • Содержит хеширование паролей(PBKDF2)

Использование любых шаблонизаторов для ваших представлений

С версии 1.1 Parser пакет поставляется с FuelPHP, что позволяет использовать любой шаблонизатор(шаблон-анализатор) Включены следующие библиотеки:

  • Mustache
  • Markdown
  • Smarty
  • Twig
  • Haml
  • Jade
  • Dwoo
  • Phptalhttp://particletree.com/features/php-quick-profiler/

Скопируйте в папку Parser необходимый шаблонизатор, включите пакет Parser, и вперед!
